# Unreleased Game Research Skill

一个面向热门未上线游戏研究的 Agent Skill。

它的目标不是简单抓数据，而是帮助研究者系统理解：

- 用户到底在期待什么
- 产品的真实卖点和风险点在哪里
- 官方宣发、广谱受众、内测玩家、专业创作者之间，观点是否一致
- 这款游戏在所属品类中的生态位、潜力和潜在问题是什么

当前仓库中的 Skill 主体位于 `unreleased-game-research/`。

---

## 为什么要做这个 Skill

对于热门未上线游戏，行业里常见的问题是：

- 信息分散在官网、采访、PV、评论区、TapTap/B 站页面、创作者评测里
- 不同来源的信号噪声很大，容易被单一声音误导
- 只看宣发会高估产品，只看评论区又容易被极端情绪带偏
- 很多研究最终停留在“摘录观点”，而不是形成结构化判断

这个 Skill 的核心价值，就是把这些分散且噪声很高的信息，整理成一套可审计、可复盘、可比较的研究流程，辅助理解：

> 用户对某一热门品类未上线游戏，到底抱着什么期待；
> 这些期待是否被产品的当前测试版本所承接；
> 市场真实反馈中，哪些是长板共识，哪些是风险预警。

---

## 整个项目的研究逻辑

这个 Skill 将未上线游戏研究拆成四个层次，分别去“倾听”不同类型的人。

### Section 1：倾听媒体的声音

这一层看全球范围内的公开情报。

关注的问题包括：

- 游戏是是哪个工作室/团队在做，这个团队历史上做过什么产品
- 制作人/主创有没有公开信息
- 从首曝到最新测试经历了几轮测试
- 每一轮测试更新、调优、重做了什么
- 游戏从公开信息看，真实卖点、玩法结构、品类定位是什么

这一层的作用，是给整个研究建立“产品理解底座”。

### Section 2：倾听广谱玩家 / 受众的声音

这一层关注官方视频下面的大规模评论反馈，例如：

- YouTube 官方视频评论
- B 站官方 PV / 实机演示评论

这一层反映的是更广谱、更大众、更偏第一印象的声音。

它能回答：

- 玩家第一眼被什么吸引
- 大家最先担心什么
- 不同阶段 PV / 实机放出后，情绪和关注点如何变化

### Section 3：倾听拿到内测资格的玩家的声音

这一层关注的是主页评论、预约页评论、长评等反馈。

这些人本质上是：

- 真正接触到了当前测试版本的人
- 在一定程度上经过官方筛选的优质玩家
- 相比评论区路人，更接近真实体验者

这一层能够帮助判断：

- 宣发中的卖点落地得怎么样
- 玩家真正玩过之后，最在意的问题是什么
- 美术、战斗、养成、剧情、商业化、优化等维度里，哪些是真的被认可，哪些是“宣发强、落地弱”

### Section 4：倾听更专业的游戏玩家的声音

这一层关注的是创作者评测视频。

这些创作者通常：

- 具备更丰富的游戏经历
- 会主动做竞品比较
- 更擅长拆解系统、体验、节奏、结构和市场定位
- 往往把游戏评测当作副业或主业

所以，Section 2 -> Section 4 的样本量是在下降的，但对单个样本的分析深度是在上升的。

## 整体执行逻辑

这个 Skill 的执行逻辑可以概括为：

1. 数据采集
2. 结构化落盘
3. 分 Section 分析
4. 跨 Section 综合推理
5. 生成 executive summary 和 final report

---

## 每个 Section 的数据采集 -> 分析链路

### Section 1：公开情报研究

链路：

`Agent 全网搜索 -> 汇总可验证来源 -> 生成结构化事实 -> 基于分析框架输出 findings`

### Section 2：官方视频评论分析

链路：

`采集 YouTube/B 站官方视频评论 -> 清洗/标准化 -> 情绪与话题归类 -> 生成 findings + evidence tables`

### Section 3：主页与长评分析

链路：

`抓取 TapTap / B 站主页或评论页 -> 筛出长评/高价值评论 -> Agent 深度归纳 -> 输出结构化 findings`

### Section 4：创作者评测分析

链路：

`筛选评测视频 -> 下载字幕或音频 -> 火山引擎 ASR 转录为 txt -> Agent 深读逐字稿 -> 输出深度 findings`

---

## 最终产出是什么

最终会形成两类核心结论：

### 1. Section Findings

每个 Section 都会生成自己的 `findings.md`，回答对应层级的问题。

### 2. 综合报告

最后会生成：

- `synthesis/executive_summary.md`
- `synthesis/final_report.md`

其中：

- `executive_summary.md` 更像管理层摘要
- `final_report.md` 更像完整的分析师报告

## 如何使用这个 Skill

### 1. Skill 主体位置

Skill 主体位于：

`unreleased-game-research/`

核心文件包括：

- `unreleased-game-research/SKILL.md`
- `unreleased-game-research/scripts/`
- `unreleased-game-research/references/`
- `unreleased-game-research/assets/`

### 2. 适合什么场景

当你希望研究一款热门未上线游戏，并回答以下问题时，可以使用这个 Skill：

- 用户到底期待什么
- 官方宣发是否有效
- 测试玩家与创作者的真实态度是什么
- 这款产品在所属品类里有没有机会跑出来

### 3. 基本使用方式

如果你的 Agent 支持 Skill 调用，直接让 Agent 使用 `unreleased-game-research` 即可。

### 4. 运行方式

完整研究流程通常是：

1. 初始化项目
2. 补齐凭证 / 页面链接 / 候选视频列表
3. 运行各 Section
4. 让 Agent 进行综合分析和报告生成

你可以从 `unreleased-game-research/SKILL.md` 和 `unreleased-game-research/references/research_workflow.md` 开始了解详细工作流。

### 5. Section 4 的特殊说明

Section 4 默认使用火山引擎 ASR 做视频转录，原因是：

- 配置简单
- 不依赖本地 GPU
- 成本低
- 有免费额度
- 适合中文长视频评测转录
